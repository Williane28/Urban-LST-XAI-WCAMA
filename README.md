# Machine Learning-Based Spatial Modeling of Land Surface Temperature in Butantã: Insights from SHAP Interpretability

This repository contains the source code, methodologies, and supporting materials developed for the paper **"Machine Learning-Based Spatial Modeling of Land Surface Temperature in Butantã: Insights from SHAP Interpretability"**, presented at the **17th Workshop on Applied Computing for Environmental Management and Natural Resources (WCAMA 2026)**.

The project focuses on predictive modeling of **Land Surface Temperature (LST)** in urban areas, using the Butantã district in São Paulo, Brazil, as a case study. The proposed approach employs **Gradient Boosting** algorithms, specifically **XGBoost**, combined with **Explainable Artificial Intelligence (XAI)** techniques, particularly **Shapley Additive Explanations (SHAP)**, to improve model interpretability and identify the main factors influencing LST.

## 🧠 Methodology Overview

The Machine Learning pipeline followed the methodology described in the paper, with all steps executed **independently and in parallel for the 2018 and 2021 datasets**.

1. **Data Acquisition and Integration:** Independently loaded the 2018 and 2021 datasets, integrated the corresponding external variables, and computed the *Topographic Position Index* (TPI) for each year.
2. **Data Preparation and Cleaning:** Removed records without the target variable (LST), discarded non-informative columns (e.g., dates and identifiers), and standardized variable names.
3. **Feature Selection:** Reduced multicollinearity using **Pearson correlation** with a threshold of **0.90**, retaining the variables with the strongest correlation to LST.
4. **Baseline Model Comparison:** Performed **10-fold cross-validation** on the complete dataset to compare the stability and average predictive performance of **XGBoost** and **LightGBM**.
5. **Hyperparameter Optimization:** Optimized the **XGBoost** model using **Optuna** with **5-fold cross-validation**, exploring a comprehensive search space that included tree depth, regularization parameters, and learning rates.
6. **Final Model Training and Evaluation:** Built the final model using the **holdout** strategy, splitting the data into **80% for training** and **20% for testing**. The independent test set was used to evaluate the model's generalization performance using **R², MAE, RMSE, and MAPE**.
7. **Explainability (XAI):** Applied **SHAP (Shapley Additive Explanations)** using the `TreeExplainer` algorithm. SHAP values were computed exclusively on the **20% test set** to ensure an unbiased interpretation of the final model.

## ⚠️ Note on Reproducibility and Environment Variations

The results and visualizations generated by this repository (executed locally via VS Code) may present slight visual differences compared to the exact figures published in the original paper (executed via Google Colab).

These differences may occur due to natural variations in random seed initialization, floating-point processing across different hardware and operating systems, and specific versions of the libraries used (*scikit-learn, xgboost, and shap*).

It is important to note that:

* **The overall performance metrics (R², MAE, RMSE) remain consistent** and statistically similar to those reported in the paper, confirming the stability of the models' predictive performance.

## 🚀 How to Reproduce the Study

1. **Clone the repository:**
   ```bash
   git clone https://github.com/Williane28/Urban-LST-XAI-WCAMA.git
   cd URBAN-LST-WCAMA
   ```

2. **Create and activate a virtual environment (recommended):**
   ```bash
   # On Windows:
   python -m venv .venv
   .venv\Scripts\activate

   # On Linux/macOS:
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. **Install the required dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Prepare the data:**

   Ensure that the input CSV files specified in `config.py` are correctly placed in the `data/` directory.

5. **Run the main pipeline:**
   ```bash
   python main.py
   ```

   *All outputs—including distribution plots, feature importance rankings, cross-validation results, final evaluation metrics, and SHAP artifacts—will be generated and automatically saved in the `output/` directory.*
